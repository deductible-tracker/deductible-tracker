# Introduction

Design patterns are reusable solutions to recurring problems in software engineering. Rust's combination of ownership, borrow checking, strong type system, and functional features means many classical patterns look different in Rust; some patterns are unnecessary while new, Rust-specific patterns emerge.

This documentation collects idioms, design patterns, anti-patterns, and functional usage notes for Rust. It is organized for maintainability and consumption by AI tools: each file is focused on a topic and uses consistent headings.

Why this split?

- Small, focused files are easier for tools to index and for contributors to update.
- Each section follows a consistent structure: Description, Motivation, Example, Advantages, Disadvantages, See also.
