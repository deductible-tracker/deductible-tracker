Rust Design Patterns

This folder contains the Rust Design Patterns documentation split into focused, maintainable files suitable for consumption by humans and AI-driven development tools.

Files:

- [01-Introduction](01-introduction.md) — Overview and motivation
- [02-Idioms](02-idioms.md) — One file per idiom, located in `idioms/`
- [03-Design-Patterns](03-design-patterns.md) — One file per pattern, located in `patterns/`
- [04-Anti-Patterns](04-anti-patterns.md) — Known anti-patterns and pitfalls (consider per-item files under `anti-patterns/`)
- [05-Functional-Usage-and-Optics](05-functional-usage.md) — Functional approaches and optics
- [06-Resources-and-links](06-resources.md) — Further reading and references

Subfolders:

- `idioms/` — individual files for each idiom (e.g., `02-01-borrowed-args.md`)
- `patterns/` — individual files for each design pattern (e.g., `03-01-command.md`)

Guidelines

- Files are organized by topic and intended to be machine-friendly: keep each file focused and use consistent headings (Description, Motivation, Example, Advantages, Disadvantages, See also).
- When adding new items, create a separate subsection or file under `rust/` so AI agents can index small, topic-focused documents.
